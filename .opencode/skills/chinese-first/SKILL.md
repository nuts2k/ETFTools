# Chinese First (中文优先原则)

## 核心原则

在所有输出中（包括对话、文档、注释），**默认且优先**使用中文。仅在以下情况保留英文：
1.  **代码关键字**：变量名、函数名、类名、保留字等。
2.  **专有名词**：特定的技术术语（如 `React`, `Docker`, `Payload`），如果业界通用且无习惯性中文译名，可保留英文。
3.  **路径与文件名**：系统绝对路径、相对路径、文件名。
4.  **特定引用**：引用官方英文文档原文时。
5.  **Git 提交信息**：遵循项目现有规范（通常保持英文），除非用户明确要求使用中文。

## 具体执行规范

### 1. 沟通与解释
- **拒绝中英夹杂**：避免使用 "Create 一个 new file" 这种表达，应使用 "创建一个新文件"。
- **翻译习惯**：
    - `Argument` -> 参数
    - `Return` -> 返回/返回值
    - `Deploy` -> 部署
    - `Build` -> 构建
    - `Run` -> 运行

### 2. 文档写作 (Markdown)
- 标题、正文描述必须使用中文。
- 列表项、表格内容除代码引用外，必须使用中文。

### 3. 代码注释
- **Docstring/Javadoc**：使用中文描述函数功能、参数意义和返回值。
- **行内注释**：使用中文解释逻辑。
